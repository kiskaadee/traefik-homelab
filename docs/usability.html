🔑 Authentication (SSO)

Most services are protected by Authelia.

  1. Access a protected service (e.g., gitea.*).
  2. You will be redirected to the Auth Portal (auth.*).
  3. Log in with your credentials.
  4. Once authenticated, you have global access to all SSO-enabled services.

🧭 Navigating the Hub

The Homepage dashboard is your central entry point:

  • Access URL: https://arch-services.mywire.org
  • Widgets: View real-time Docker container statuses, system resources, and service links.

Local Access (Fast Lane)

To avoid traversing the public internet when you are on the same network as the server, we use a Public Domain + Local IP strategy (see ADR 001).

Add the following to your machine's /etc/hosts file:

127.0.0.1  traefik.arch-services.mywire.org auth.arch-services.mywire.org jellyfin.arch-services.mywire.org gitea.arch-services.mywire.org excalidraw.arch-services.mywire.org homepage.arch-services.mywire.org mermaid.arch-services.mywire.org ollama.arch-services.mywire.org learning.arch-services.mywire.org logs.arch-services.mywire.org portainer.arch-services.mywire.org arch-services.mywire.org

Note: This ensures valid SSL and working SSO cookies for local sessions.

🛠️ Service Management

Deployment Scripts

# Start everything
./up.sh

# Start ONLY the Control Plane
./up.sh --infra-only

# Stop all services
./down.sh

➕ Adding a New Application

  1. Create apps/your-app/docker-compose.yml.
  2. Connect to proxy-net.
  3. Add Traefik labels for routing and Authelia protection.
  4. Update apps/homepage/config/services.yaml.

🤖 Ollama API & Automation

Uses a hybrid security model for browser and CLI access.

API Key / Trusted Payload

Header: X-Ollama-Key: sk-ollama-hardened-hub-2026

curl https://ollama.arch-services.mywire.org/api/generate \
  -H "X-Ollama-Key: sk-ollama-hardened-hub-2026" \
  -d '{"model": "llama3", "prompt": "hi"}'
