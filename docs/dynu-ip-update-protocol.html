📡 Dynu IP Monitor & Updater

Design and architecture of the DDNS synchronization mechanism.

Problem Definition

Maintain accurate DNS A records despite dynamic public IPs and unreliable providers.

Core Design Decisions

1. Probe Model

Runs periodically via systemd instead of a long-lived daemon.

2. Dual-Source

Uses DNS (primary) and HTTP (fallback) for IP discovery.

3. Validation

Strict pipeline ensures no RFC1918 or private IPs are ever synced.

4. Atomicity

State updates use atomic file moves to prevent corruption.

Failure Model

  • Degraded: Fallback used, system remains functional.
  • Error: No valid IP found, abort mutation.
