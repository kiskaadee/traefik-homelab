🗺️ Project Roadmap

Current system maturity and future priorities for the core infrastructure.

Phases

  • Phase 1: Core
  • Phase 2: Observability
  • Phase 3: Automation
  • Phase 4: Optimization

✅ Phase 1: Core Gateway & Infrastructure

COMPLETE Stable external access, identity management, and core security.

  • [x] Dynamic DNS: IP synchronization migrated to host NixOS configuration (nixos-config).
  • [x] Edge Routing: Traefik v3 with Let's Encrypt (DNS-01) automation.
  • [x] Layered Architecture: Separation of control plane and decoupled data plane folders.
  • [x] Identity & SSO: Authelia integration with 2FA/OIDC support.
  • [x] Security Hardening: Docker socket proxy and internal network segmentation.

🏗️ Phase 2: Observability & Portal Layer

IN PROGRESS Expand utility and improve system monitoring.

  • [x] Service Discovery: 'Homepage' dashboard integration.
  • [ ] Unified Health: Integrate Traefik/Docker health checks into Homepage via socket-proxy.
  • [ ] Resource Monitoring: Add lightweight system metrics (Glances/Netdata) to the portal.

🚀 Phase 3: Automation & Resilience

  • [x] Automated Updates: Watchtower and Diun for image lifecycle.
  • [ ] Backup Strategy: Automated, encrypted off-site backups for persistent core volumes.
  • [ ] Infrastructure as Code: NixOS declarative system module mapping.
  • [ ] Alerting: Push notifications (Gotify/Ntfy) for core system failures.

🌐 Phase 4: Network Optimization

  • [ ] Local DNS: Ad-blocking DNS (Pi-hole/AdGuard Home) for LAN.
  • [ ] Hairpin NAT: Validation and optimization for internal LAN routing.
  • [ ] VPN Integration: WireGuard-based secure remote access.
