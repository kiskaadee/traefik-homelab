The homeserver follows a Layered Architecture designed for security, observability, and modularity. It is split into two logical tiers: the Control Plane (Infrastructure) and the Data Plane (Applications).

1. Physical & Host Layer

The foundation of the system running directly on the host hardware or virtualized OS.

  • OS: Linux (Generic).
  • IP Sync: [DEPRECATED/MOVED] Dynu DDNS updates are now handled declaratively by the host's NixOS configuration repository (nixos-config).
  • Docker Engine: Orchestrates all containerized services.

2. Infrastructure Layer (Control Plane)

The management tier that handles routing, identity, and system maintenance.

🌐 Edge Gateway

Traefik v3: Reverse Proxy & SSL (DNS-01 Challenges). Routes subdomains for all core and user-deployed services.

🔐 Identity

Authelia: SSO & 2FA provider. Forward Auth integration with Traefik.

🛡️ Socket Proxy

Protects the Docker socket from container-escape vulnerabilities.

🛠️ Maintenance

Portainer (Management), Watchtower (Updates), and Diun (Notifications).

3. Application Layer (Data Plane)

Consumer-facing services isolated from the core infrastructure.

  • Service Discovery: Homepage dashboard serves as a unified entry portal.
  • SSO Protection: Critical administration pages and external services utilize Authelia Forward Auth.
  • Decoupled Lifecycles: User applications run in separate containers outside this repository, keeping the core system clean and stable.
  • Isolation: Services have no direct host access and communicate strictly over dynamic internal networks.

4. Networking Model

Traffic flow is segmented to minimize the blast radius of a potential breach.

  • proxy-net: Shared internal network for Traefik and all exposed applications.
  • socket-net: Isolated network for Docker management tools.
  • External: Only ports 80 and 443 are exposed.

🛡️ Security Posture

Current hardening measures enforced across the Hub:

  • HARDENED No Root Docker: Standardized PUID/PGID usage.
  • PRIVATE Wildcard SSL: Prevents subdomain enumeration via certificate transparency logs.
  • ISO Network Segmentation: Strict isolation of management traffic.
